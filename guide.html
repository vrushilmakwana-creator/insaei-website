2

Create a GitHub Repository

GitHub will host your files for free. You need a free account at github.com to get started.

  1. Sign in to GitHub Go to github.com → click your avatar (top right) → Your repositories → New.
  2. Name your repository Use something like insaei-website or, for the cleanest URL, name it yourusername.github.io (replace with your actual GitHub username). This makes it your personal site root.
  3. Set visibility to Public GitHub Pages only works on Public repos on free accounts. Leave "Add a README" unchecked — you'll upload your own files.
  4. Click "Create repository" You'll land on an empty repo page. Keep this tab open.
  5. Upload your files Click uploading an existing file (shown on the empty repo page). Drag all files from your project folder into the box, then click Commit changes. Make sure index.html is at the root level, not inside a subfolder.
⚠ Important All files must be at the root of the repository — not inside a folder like /website/. GitHub Pages serves index.html as the homepage automatically.
3

Enable GitHub Pages

  1. Go to repository Settings Inside your repo, click the Settings tab (gear icon, far right of the top nav).
  2. Open Pages settings In the left sidebar scroll to Code and automation → Pages.
  3. Set the source Under Source, choose Deploy from a branch. Set Branch to main and folder to / (root). Click Save.
  4. Wait ~60 seconds GitHub builds and deploys your site. Refresh the Settings → Pages page until a green banner appears showing your live URL.
Your live URL will look like https://yourusername.github.io/insaei-website/

If you named the repo yourusername.github.io the URL will simply be https://yourusername.github.io — no subfolder.

4

Create an EmailJS Account

EmailJS lets a static site send real emails without a back-end. The free tier gives you 200 emails/month which is plenty to start.

  1. Sign up at emailjs.com Go to emailjs.com → Sign Up Free. Use your Gmail address.
  2. Note your Public Key After logging in go to Account → API Keys. Copy the Public Key — it looks like aBcDeFgHiJkLmNoP. You'll paste this into your HTML.
5

Connect your Gmail Service

  1. Go to Email Services In the EmailJS dashboard click Email Services in the left sidebar → Add New Service.
  2. Choose Gmail Select the Gmail tile. Give it a friendly name like innsaei_gmail.
  3. Connect your Google account Click Connect Account. A Google OAuth window opens. Allow access. EmailJS will send emails from your Gmail address.
  4. Send a test email Click Send Test Email. Check your Gmail inbox — if it arrives you're good. Click Create Service.
  5. Copy your Service ID After saving, you'll see a Service ID like service_abc1234. Copy it.
6

Build your Email Template

A template defines what the email looks like when it lands in your inbox.

  1. Create a new template Click Email Templates → Create New Template.
  2. Set the Subject line Use: New Enquiry from {{from_name}} — {{service}}
  3. Write the Body Paste the template below into the body editor. The {{variable}} placeholders are filled automatically from the form.
  4. Set Reply-To In the Reply To field put {{reply_to}} so you can reply directly to the person who contacted you.
  5. Save and copy Template ID Click Save. Your Template ID looks like template_xyz9876. Copy it.

Paste this into the EmailJS body editor:

You have a new enquiry via the INNSÆI website.

─────────────────────────────────
  Name      : {{from_name}}
  Email     : {{from_email}}
  Phone     : {{phone}}
  Company   : {{company}}
  Service   : {{service}}
─────────────────────────────────

Message:
{{message}}

─────────────────────────────────
Reply directly to this email to respond to {{from_name}}.

Template variables used — these must match exactly:

{{from_name}}Full name from form
{{from_email}}Sender's email
{{phone}}Phone number
{{company}}Company / Organisation
{{service}}Selected service
{{message}}Tell Us More textarea
{{reply_to}}Used in Reply-To field

Wire up your IDs in the code

Open the contact.html file (the updated one provided) and find the three ★ Replace markers. There are only three values to change:

/* In <head> — initialize EmailJS */
emailjs.init({ publicKey: "YOUR_PUBLIC_KEY" });
                         ↑ paste your Public Key from Account → API Keys


/* In the form submit handler */
await emailjs.send(
  'YOUR_SERVICE_ID',   // ← e.g. 'service_abc1234'
  'YOUR_TEMPLATE_ID',  // ← e.g. 'template_xyz9876'
  templateParams
);
🔍 How to find each value Public Key → EmailJS dashboard → Account → API Keys
Service ID → Email Services → click your Gmail service → Service ID
Template ID → Email Templates → click your template → Template ID (shown in the top bar)

After editing, save the file and re-upload it to your GitHub repo (drag-and-drop the updated file, commit). GitHub Pages redeploys automatically within ~30 seconds.

8

Test & Go Live

  1. Visit your live GitHub Pages URL Open https://yourusername.github.io/insaei-website/contact.html in a browser (not locally from your computer — it must be the live URL).
  2. Fill in the contact form Use a real name, your own email, pick any service, type a short message. Click Send Message.
  3. Watch for the success state The form should hide and show the "Message received." panel. This means EmailJS accepted the request.
  4. Check your Gmail The formatted email should arrive within a few seconds. Hit Reply — it will go to the address the visitor typed in the form.
  5. Check EmailJS dashboard logs Go to EmailJS → Email History to see a log of all sends. Useful for debugging if an email doesn't arrive.
⚠ Testing locally won't work EmailJS only allows sends from domains you've whitelisted. Go to EmailJS → Account → Security and add your GitHub Pages domain (e.g. yourusername.github.io) to the allowed origins. Without this, browsers will block the request.
9

Custom .com Domain (Optional)

When you're ready to point a real domain (e.g. innsaei.com) to your site:

  1. Buy a domain Use Namecheap, GoDaddy, Google Domains, or Cloudflare Registrar. Cloudflare is cheapest for renewals.
  2. Add a CNAME record In your domain registrar's DNS settings, add a CNAME record: www → yourusername.github.io. For the apex domain (innsaei.com without www) add four A records pointing to GitHub's IPs: 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153.
  3. Enter custom domain in GitHub In repo Settings → Pages → Custom Domain, type www.innsaei.com. GitHub will verify DNS (takes up to 48 hours to propagate).
  4. Enable HTTPS Once DNS is verified, tick Enforce HTTPS in the same Settings → Pages panel. GitHub auto-provisions a free SSL certificate via Let's Encrypt.
  5. Update EmailJS allowed origins Go to EmailJS → Account → Security and add your new domain (https://www.innsaei.com) alongside the GitHub Pages URL.
